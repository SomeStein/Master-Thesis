
\chapter{Numerical Testing}
\label{ch:numericalTesting}

This chapter compares the invariant-polytope algorithm (Algorithm~\ref{alg:invariantPolytope}), the finite-tree algorithm (Algorithm~\ref{alg:finiteTree}), and the invariant-tree algorithm (Algorithm~\ref{alg:hybridApproach}) with respect to iteration counts, runtime, and certificate quality.

\section{Experimental Setup}
\label{sec:numericsExperimentalSetup}
This chapter is intended to document the computational evidence for the thesis. In its final form, it should make clear which matrix families were tested, how the algorithms were implemented, and which quantitative criteria were used for the comparison.

All experiments should be reported with the same preprocessing pipeline from Section~\ref{sec:introPreprocessing}. In particular, the candidate-selection strategy, the scaling convention, the stopping tolerances, and the maximal iteration counter should be kept fixed unless a section explicitly studies their effect.

The numerical comparison should at least record the following quantities:
\begin{itemize}
    \item whether the algorithm terminated with an exact certificate,
    \item the number of outer iterations until termination or stopping,
    \item the wall-clock runtime,
    \item the size of the produced certificate object, such as the number of vertices or tested leaves,
    \item and, in the non-terminating case, the interval $[\underline{\rho},\overline{\rho}]$ returned for the JSR.
\end{itemize}
