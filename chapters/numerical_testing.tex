
\chapter{Numerics}
\label{ch:numericalTesting}

This chapter compares the invariant-polytope algorithm (Algorithm~\ref{alg:invariantPolytope}), the finite-tree algorithm (Algorithm~\ref{alg:finiteTree}), and the invariant-tree algorithm (Algorithm~\ref{alg:hybridApproach}) with respect to iterations, runtime, and solution quality.

\section{Solves on generated Matrix Sets}
\label{sec:numericsGeneratedSets}
In this section we compare all three algorithms on generated matrix families across increasing dimensions. The focus is quantitative scalability with respect to dimension and number of tested instances.

\section{Solved Problems with One s.m.p.}
\label{sec:numericsSingleSmpSpeed}
This section reports comparisons on benchmark families with a unique s.m.p., with iteration count (steps to termination) as the primary speed criterion.

\section{Solved Problems with Multiple s.m.p.s}
\label{sec:numericsMultiSmpQualitySpace}
This section compares the effective solve space for multiple-s.m.p.\ instances, i.e., which classes of problems can be solved reliably and with what certificate quality by the new algorithm.
